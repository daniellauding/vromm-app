## Design Principles

### Visual Identity
- **Friendly & Approachable**: AI should feel like a helpful companion, not a robot
- **Non-Intrusive**: Always accessible but never dominating the interface
- **Contextual**: Adapts appearance based on where user is in the app
- **Consistent**: Follows Vromm's existing design system (Tamagui)

### Color Palette
```
AI Brand Colors:
- Primary: #4A90E2 (Friendly Blue - trust, knowledge)
- Secondary: #7B68EE (Purple accent - intelligence)
- Success: #5CB85C (Green - encouragement)
- Alert: #F0AD4E (Warm orange - gentle nudges, not errors)
- Background: #F8F9FA (Light grey - subtle, not white)
- Text: #2C3E50 (Dark grey - readable)
```

### Typography
```
- AI Messages: System font, 16sp (readable, conversational)
- User Messages: Same font, 16sp (equality in conversation)
- Quick Actions: 14sp, Medium weight
- Timestamps: 12sp, Light weight, Grey
```

### Spacing & Layout
```
- Message bubbles: 16dp padding vertical, 12dp horizontal
- Bubble margin: 8dp between messages
- Screen margins: 16dp (mobile), 24dp (tablet)
- FAB padding from edge: 16dp bottom, 16dp right
```

## Responsive Design

### Mobile (Phone - Portrait)
- Chat modal: 75% screen height
- Route cards: Full width (minus 16dp margins)
- Quick actions: 3 visible, horizontal scroll
- FAB: Bottom-right corner

### Mobile (Phone - Landscape)
- Chat modal: 60% screen height (less vertical space)
- Message bubbles: Max 70% width (more horizontal space)
- Compact header (40dp instead of 56dp)

- Side-by-side: Map on left, chat on right (split screen mode)

### Tablet (iPad - Landscape)
- Chat modal: Docked to right side (40% screen width)
- Main app on left (60% width)
- FAB in bottom-right of app area (not chat)
